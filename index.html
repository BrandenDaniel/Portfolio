Foodie Hotspots

2018
launch sitegithub repo
Built with:
Project status: complete

Foodie Hotspots (named by me), was one of my final challenges within the completion of Udacity's front-end course.

This nifty little project turned out to look better than expected. The design was highly apprasied by my project reviewer, who commented "This is the best design I have seen so far."

More on the concept - Foodie Hotspots is a cool utility website where users can search for reviews of restaurants within a given coordinate. Information like star ratings and comments will be displayed.

Being only a concept, as of now the database of restaurants is only a hardcoded json file. However perhaps in the near future, I would love to turn this concept into a real life functional web-application via an API such as Yelp's.

Reddit Searcher

2018
launch sitegithub repo
Built with:
Project status: complete

This simple web-app allows you to search for any desired topic within the reddit API database.

Although a simple concept, it was indeed enjoyable constructing a dynamic and clean UI.

Of course, the heart of the Reddit searcher app is Javascript. It is fully coded in vanilla Javascript and utilised with ES6's syntax.

Digging through APIs and explore the data they could offer can be really fun, especially when working with a diverse API such as Reddit's. As of current, a searched post will display it's subreddit, score, and comments amount.

Matching/Memory Game

2018
launch sitegithub repo
Built with:
Project status: complete

The Matching/Memory Game is your classic card flip game that requires full concentration, and a little luck.

This project was another's by Udacity front-end Nanodegree program. It was the first time I've built a fully functional game application from the ground up.

Initially the Javascript was a little tricky, I had to figure out the logics of temporarily storing variables within arrays that will be overwritten upon a user's every new guess.

The styling aspect, however, was really fun and challenging to create. After tons of adjustments and experiments, I went with a game-ish theme to suit the purpose of the app, and really try to draw in the user with the interfaces' bright colours.

Frogger Simulator (Desktop Game)

2018
launch sitegithub repo
Built with:
Project status: complete

The Frogger Simulator emulates the style of the very popular Frogger game.

This project was undertaken while I was completing Udacity's front-end Nanodegree course. It was my first JS intensive project to complete which encouraged me to think outside of the box & explore JS and it's relations to the DOM in different levels.

The learning curve was quite steep for me in this endeavour of coding. I found it exceedingly difficult to grasp the connection between certain functions & logics. It was a lot of research and patience that led me to complete the frogger simulator.

Users can interact by collecting randomly generated star points or lose points when intersected with a roach. As of now the game only works with keyboard inputs.

E-commerce Theme Website

2017
launch sitegithub repo
Built with:
Project status: not complete

This project was a fun experiment where my goal was to construct an e-commerce website with attractive UI components.

The website is constructed with some Bootstrap components, a little of Javascript, and custom CSS. The process began when I became inspired by awesome-looking websites, (ASOS and THE ICONIC to mention a few) and decided to see if I could make one myself.

The bulk of the work is at the custom CSS. The main goal when persuing my own e-commerce website was to neatly group components and highlight what I thought was important to the user's attention. What I learned from the many feats of challenges was that even the tiniest of details could make or break the whole user experience.

Admittedly the site is still far from a real life fully functional e-commerce website, however slow improvements are made when seen available.
